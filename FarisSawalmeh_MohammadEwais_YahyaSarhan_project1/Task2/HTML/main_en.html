Network Switching Techniques

Circuit Switching

Circuit Switching gives a deticated "private" line between two devices. Unlike Packet switching once a connection is established, it is more reliable and less bandwidth efficient.

Because there are limited lines between devices, the consuption of bandwidth is also limited.So there is a maximum number of connections that can be established at once. There are also 2 different types of circuit switching:

  • Frequency Division Multiplexing
  • Time Division Multiplexing

Packet Switching

Packet Switching is the process of breaking down a file into smaller parts called packets and sending them through multiple links instead of the file itself. Although this may seem simpler than sending an entire file at once, we run into some slight problems, facing different types of delays:

  1. Transmission Delay
  2. Propogation Delay
  3. Queueing Delay

In packet switching the packages are sent from router to router over links where each router contains these delays. In order to send the file through multiple routers we need to make sure that all the packets are recieved in one router for it to move to another.
